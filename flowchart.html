The Left-to-Right Transition Mechanism

Rather than traditional separate pages, the site uses a hardware-accelerated wrapper container layout with overflow-hidden on the root window. Each panel takes up exactly 100vw (100% of the viewport width). Inside the navigation controller, clicking a menu link or swiping translates the container horizontally along the X-axis:

// Smooth left-to-right CSS Transition rules
.slide-container {
  display: flex;
  width: 500vw;
  transition: transform 0.7s cubic-bezier(0.4, 0, 0.2, 1);
}

Mobile Gesture Support & Responsiveness

To ensure optimal UX on touch devices, swipe detection is registered via Javascript. Left-to-right page movements are triggered dynamically by tracking:

  • touchstart: Record initial horizontal coordinates.
  • touchend: Calculate swipe direction and force offset thresholds.
  • Automatically transitions adjacent panels if threshold is exceeded.
